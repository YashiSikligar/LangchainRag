### RAG Pipeline using LangChain

This document explains the implementation of a Retrieval-Augmented Generation (RAG) system using LangChain, step by step.

#### Class: RagLangchain

We create a class named RagLangchain that handles the entire RAG pipeline — from loading a document to generating answers based on retrieved context.

#### Constructor

The constructor initializes the following variables:

| Variable      | Description                                     |
|---------------|-------------------------------------------------|
| path          | Path/location of the file to be loaded         |
| document      | Holds the loaded document (initially None)     |
| chunks        | Stores sentence-wise chunks of the document   |
| vector_store  | Stores embeddings of the chunks for fast retrieval |
| retriever     | FAISS search retriever for fetching relevant chunks |
| llm           | LLM model used to generate answers            |
| prompt        | Instruction template guiding the LLM to answer logically |

#### Method 1: file_open

 - Purpose: Load the document (PDF, text, images, tables).

 - Implementation: Uses PyMuPDFLoader to load the file.

 - Steps:

  1. Load file using the given self.path.

  2. Extract page content and metadata into self.document.

  3. Return self.document.

- PyMuPDFLoader is useful for documents containing text, images, and tables.

#### Method 2: text_splitter

- Purpose: Split the document into smaller, manageable chunks.

- Implementation: Uses RecursiveCharacterTextSplitter.

- Parameters:

1. chunk_size=200 → Each chunk has ~200 characters.

2. chunk_overlap=50 → Each chunk overlaps 50 characters from the previous chunk.

- Steps:

  1. Split document into chunks while retaining metadata.

  2. Store chunks in self.chunks.

  3. Return self.chunks.

#### Method 3: embedding

- Purpose: Create embeddings for each chunk for semantic search.

- Implementation: Uses OpenAI embedding model text-embedding-3-small.

- Steps:

  1. Generate embeddings for self.chunks.

  2. Store embeddings in FAISS vector store (self.vector_store).

  3. Return self.vector_store.

- FAISS stores embeddings efficiently for fast similarity search.

#### Method 4: retrieval

- Purpose: Retrieve relevant chunks for a given query.

- Implementation:

  1. Uses self.vector_store for similarity search (k=9).

  2. Assigns LLM for generating answers.

  3. LLM Setup:

  4. Model: gpt-4o-mini

  5. Temperature: 0.2

  6. Low temperature → factual answers

  7. High temperature → more random or creative answers

  8. Prompt: Guides the LLM to answer only based on the provided context and avoid hallucinations.

#### Method 5: model_answer

- Purpose: Generate answers for a list of questions (test_set).

- Steps:

  1. Initialize empty lists:

  2. list_question, model_answer, ground_answer, retrieved_answer

  3. Loop through each question in test_set:

  4. Append question to list_question.

  5. Append actual answer to ground_answer.

  6. Use retriever to fetch relevant chunks and append to retrieved_answer.

  7. Create context_text by joining retrieved chunks.

  8. Pass context_text and question to the LLM via the prompt.

  9. Extract answer from result.content and append to model_answer.

- Return:

+ list_question → List of questions

+ model_answer → Answers generated by LLM

+ ground_answer → Actual answers

+ retrieved_answer → Retrieved relevant chunks
